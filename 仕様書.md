# アキオカ発注管理システム 仕様書

> 最終更新: 2026-04-08  
> アプリケーション: akioka-web-application-pc  
> フレームワーク: Laravel 9.19 + Vue 3 (Inertia.js)

---

## 1. システム概要

株式会社アキオカの社内業務を統合管理するWebアプリケーション。発注管理を中核とし、在庫管理・承認ワークフロー・昼食管理・動画管理・サイネージ・FAX管理など多機能を統合している。

### 技術スタック

| 項目 | 技術 |
|------|------|
| バックエンド | Laravel 9.19 (PHP 8.0.2+) |
| フロントエンド | Vue 3.2.41 + Inertia.js |
| ビルドツール | Vite 4.0.0 |
| CSSフレームワーク | Tailwind CSS 3.2.1 |
| データベース | MySQL (Port 3308) |
| 認証 | Laravel Sanctum + セッションベース |
| API連携 | Google Auth, Guzzle HTTP |

### 主要フロントエンドライブラリ

- Chart.js - グラフ表示
- FullCalendar v6.1.15 - カレンダーUI
- Konva / Fabric.js - キャンバス操作
- PDF.js / html2pdf - PDF処理
- QRCode - QRコード生成
- Font Awesome 6.6.0 - アイコン

## 4. モデル一覧 (89ファイル)

全モデルは `app/Models/` に配置。主要なリレーションは以下の通り:

### 主要リレーション

```
User ─┬─ belongsTo → Group (部門)
      ├─ belongsTo → Position (役職)
      ├─ belongsTo → Process (工程)
      └─ hasMany → OrderRequest, LunchOrder, Task, etc.

Stock ─┬─ hasMany → StockSupplier → hasMany → StockSupplierPrice
       ├─ hasMany → StockStorage → belongsTo → Location, StorageAddress
       ├─ belongsTo → StockClassification
       └─ hasMany → InventoryOperationRecord

OrderRequest ─┬─ belongsTo → User
              ├─ belongsTo → Supplier
              ├─ belongsTo → Stock
              ├─ hasOne → InitialOrder
              └─ hasMany → OrderRequestApproval

InitialOrder ─┬─ belongsTo → Supplier
              └─ belongsToMany → Delivery (中間テーブル: delivery_initial_orders)

ApprovalFlow ─┬─ hasMany → ApprovalFlowStep → belongsTo → User (approver)
              └─ hasMany → ApprovalFlowCondition

ScheduleEvent ─┬─ belongsTo → Facility
               └─ hasMany → ScheduleParticipant → belongsTo → User
```

## 8. 主要機能詳細

### 8.1 承認フローシステム

条件ベースの動的承認ルーティング:

- **役職条件**: ユーザーの役職に基づく承認経路
- **部門条件**: 所属部門による分岐
- **金額条件**: price_min / price_max によるエスカレーション
- **品目種別**: 新規品目 vs 既存品目
- **複数ステップ**: step_orderによる順序制御、必須/任意の指定
- **サービスクラス**: ApprovalFlowService で条件評価ロジックを集約

### 8.2 在庫管理システム

- 品目マスタ管理（分類・画像・URL・メモ）
- 複数仕入先対応（メイン仕入先フラグ）
- 仕入先別単価管理（期間・有効フラグ付き価格履歴）
- 保管場所管理（ロケーション + 住所）
- 在庫操作記録（入庫・出庫・棚卸等のトランザクション）
- 滞留在庫監視・処置記録
- データエクスポート（CSV）
- タブレット対応UI

### 8.3 発注管理システム

- 発注依頼作成 → 承認ワークフロー → 発注 → 納品
- 消耗品注文（別ワークフロー）
- 初回発注管理（新規取引先への最初の発注）
- 納品管理（多対多: 複数初回発注 ↔ 複数納品）
- デバイスメッセージ連携（通知）
- 数量・価格のオーバーライド

### 8.4 昼食管理システム

- 毎日の昼食メニュー管理・説明登録
- ユーザー予約・注文
- 注文アーカイブ
- 月次集計・CSV出力

### 8.5 サイネージシステム

- PDFベースのコンテンツ管理
- 安全記録・点検状況・納品追跡の動的表示
- 施設スケジュール表示
- 幹部スケジュール表示

### 8.6 FAX管理システム

- FAX仕分けルール管理
- ユーザーグループ・フォルダ管理

### 8.7 動画管理システム

- 動画アップロード・管理
- 文字起こし機能
- カテゴリ・タグによる分類

### 8.8 その他の機能

- **タスク管理**: ガントチャート対応
- **デバイス管理**: IP/MACアドレス・接続状態追跡
- **監視カメラ連携**: カメラ映像統合
- **Raspberry Pi連携**: センサーデータ収集
- **温湿度モニタリング**: 環境データ記録
- **リモートPC管理**: リモートコンピュータ管理
- **ラベルデザイナー**: ラベル作成機能
- **製品計算**: 在庫計算ツール
- **安全管理**: KY(危険予知)・設備点検

## 12. 開発上の注意事項

1. **Inertia.js**: ページレンダリングはInertia経由。API不要でVueコンポーネントに直接データを渡せる
2. **認証**: MainControllerによるカスタムログイン（Laravel Breezeではない）
3. **承認フロー**: DB駆動の動的条件マッチング。新条件追加時はApprovalFlowServiceを更新
4. **タブレットUI**: 一部機能にタブレット専用インターフェースあり（StockTablet, CalcProductTablet）
5. **テーブル作成**: 多くのテーブルはマイグレーションではなく直接SQLで管理されている
6. **Ziggy**: ルート共有ライブラリ。Bladeテンプレートで`route()`ヘルパーを使用可能
